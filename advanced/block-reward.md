# Функционал наград за блок

## Общее описание

C каждым блоком помимо суммы комиссий происходит эмиссия новых монет. Сумма `block_emission_amount` - параметр в [chain_parameters](../api-reference/echo-objects/chain-parameters.md).

Награда за блок распределяется между продюсером и верифаерами после применения следующего блока, в который попадает сертификат. Сумма рассчитывается и записывается в опции аккаунта, но будет начислена на баланс во время ближайшего обслуживания.

Начисление наград происходит во время `maintenance_interval` со следующими правилами:

1. В [chain_parameters](../api-reference/echo-objects/chain-parameters.md) добавлено поле `block_producer_reward_ratio`. Следовательно, продюсеру начисляется `block_producer_reward_ratio` процентов, а верифаерам оставшаяся часть. Часть награды верифаеров распределяются между ними пропорционально их балансу. При этом, бэкапы не учитываются.

2. В случае, если сообщение было выпущено делегатом(delegate), распределение наград все равно происходит как в шаге выше, но только фиксируется процент, который указан у делегата на момент выбора его таковым. то есть при апдейте параметра и изменении делегата процент устанавливается в аккаунт делегатор. Параметр процента `active_delegate_share` в опциях аккаунта. 

3. В случае, погрешности вычислений в пропорциях, излишки начисляются продюсером.

4. Cообщения выпущенные с помощью бэкап механизма(если в сети нет ни делегата, ни верифаера, а сообщение выпустил член комитета) - не учитываться при распределении. Если сертификат состоит только из сообщений бэкапов - вся награда отдаётся продюсером.

## Используемые операции

Виртуальная операция, указывающая на выплату наград за блок - [block\_reward\_operation](../api-reference/echo-operations/block-reward.md#block_reward_operation). В которой будет сумма за весь прошлый период.
